## Goal

Compose and send a summary email of today's job matches using the agentmail API. The email groups matches by country and includes the key fields a candidate needs to decide which postings to pursue. Always include all fields — even when values are "not listed" — so the reader has a complete picture for every listing.

## Inputs

- All `.md` files in `jobMatches-Can/` and `jobMatches-US/`
- `AGENTMAIL_API_KEY` and `AGENTMAIL_SENDER` from `.env`

## Steps

1. Read all `.md` files in `jobMatches-Can/` and `jobMatches-US/`. Parse YAML frontmatter for each.

2. **If both directories are empty**, send a "no matches" email instead:
   - Subject: `Job Search Results — {YYYY-MM-DD} (0 matches)`
   - Body: "No job listings matched today's criteria."
   - Then skip to step 5.

3. Sort listings within each group by `match_score` descending.

4. Compose the email in both plain text and HTML using the templates below.

**Plain text template:**
```
Subject: Job Search Results — {YYYY-MM-DD} ({total} matches)

== Canadian Matches ({n} jobs) ==

1. {title} — {company}
   Score: {match_score}/10 | Location: {location} ({location_type})
   Salary: {salary} | Sponsorship: {sponsorship} | Clearance: {security_clearance}
   {url}

   {match_rationale}

[... repeat for each CA match ...]

== US Matches ({n} jobs) ==

[... same format ...]

---
Generated by JobSearchAgent on {date}.
```

**HTML template:**
```html
<h2>Canadian Matches ({n} jobs)</h2>
<ol>
  <li>
    <strong>{title} — {company}</strong><br>
    Score: {match_score}/10 | Location: {location} ({location_type})<br>
    Salary: {salary} | Sponsorship: {sponsorship} | Clearance: {security_clearance}<br>
    <a href="{url}">{url}</a><br>
    <em>{match_rationale}</em>
  </li>
  ...
</ol>

<h2>US Matches ({n} jobs)</h2>
<ol>
  ... same structure ...
</ol>

<hr>
<small>Generated by JobSearchAgent on {date}.</small>
```

5. Load `AGENTMAIL_API_KEY` and `AGENTMAIL_SENDER` from `.env`. Send the email via the agentmail API (see reference below). If the API returns a non-2xx status, report the status code and response body — do not silently continue.

6. Report: email sent successfully with {n} Canadian and {n} US listings.

## agentmail API Reference

Send a POST request to `https://api.agentmail.to/v0/inboxes/{AGENTMAIL_SENDER}/messages/send` with JSON body:
```json
{
  "to": ["wasabi.buff@gmail.com"],
  "subject": "...",
  "text": "...",
  "html": "..."
}
```
with header `Authorization: Bearer {AGENTMAIL_API_KEY}`.

Note: The `/messages` path (without `/send`) is for reading, not sending. The correct send path is `/messages/send`.
