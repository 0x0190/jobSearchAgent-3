## Goal

Compose and send a summary email of today's job matches using the agentmail API. The email groups matches by country and includes the key fields a candidate needs to decide which postings to pursue.

## Inputs

- All `.md` files in `jobMatches-Can/` and `jobMatches-US/`
- Agentmail API credentials from `.env`

## Steps

1. Read all `.md` files in `jobMatches-Can/` and `jobMatches-US/`. Parse YAML frontmatter for each.

2. Sort listings within each group by `match_score` descending.

3. Compose the email body in plain text + HTML:

```
Subject: Job Search Results — {YYYY-MM-DD} ({total} matches)

== Canadian Matches ({n} jobs) ==

1. {title} — {company}
   Score: {match_score}/10 | Location: {location} ({location_type})
   Salary: {salary} | Sponsorship: {sponsorship} | Clearance: {security_clearance}
   {url}

   {match_rationale}

[... repeat for each CA match ...]

== US Matches ({n} jobs) ==

[... same format ...]

---
Generated by JobSearchAgent on {date}.
```

4. Send the email via agentmail API:
   - To: wasabi.buff@gmail.com
   - Subject: `Job Search Results — {YYYY-MM-DD} ({total} matches)`
   - Body: the composed text above
   - Use the AGENTMAIL_API_KEY from `.env`

5. Report: email sent successfully with {n} Canadian and {n} US listings.

## agentmail API Reference

Use the agentmail REST API (base URL and API key from `.env` as `AGENTMAIL_API_KEY` and `AGENTMAIL_INBOX_ID`). Send a POST request to `https://api.agentmail.to/v0/inboxes/{inbox_id}/messages` with JSON body:
```json
{
  "to": ["wasabi.buff@gmail.com"],
  "subject": "...",
  "text": "...",
  "html": "..."
}
```
with header `Authorization: Bearer {AGENTMAIL_API_KEY}`.
